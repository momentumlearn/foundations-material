The Shell

The shell: a text-based user interface that gives you access to the operating system. We call that interface "the command line".

We're using a Unix shell called the Bash shell, which is the default on most Linux systems and Mac OS.

Demo opening the terminal (which is just a program that gives us the command line) and have students follow along.

It's ok for students to use their own terminal if they are on Mac or Linux, but everyone can also use glitch.com. Start a new project and then from the project menu, click "Advanced Options", then choose "Open Console".

The Prompt

The prompt is the place in the terminal where you type commands.

It usually shows your computer's name (hostname) and your username

$ Alexs-MacBook-Pro:~ alexrandall$ pwd

When commands are entered successfully, you don't always receive feedback that something happened. The usual indication that everything is OK is the appearance of the prompt.

Commands

The structure of commands follows a consistent pattern.

Options and arguments are not always required, but different commands have different usages.


            $ command [-options] [arguments]

A good example command to demo is cal. Try entering it without arguments first.

Arguments

Arguments are information included with a command to modify its behavior.

One or more arguments may usually be included, separated by whitespace.


            $ cal Nov 2018

Options


            $ cal may
              // doesn't work because the command require two arguments
            $ cal -m may
              // passing the option -m lets you use a single argument for the month

man Pages

How do you know what a command does, or what arguments and options a command can accept?

The man (short for "manual") command accepts the name of other commands as arguments.


            $ man cal

man pages use another utility called less to display information. Check out the man pages on it!

To get back to the prompt from less, press "Q".

You can demo this with any command, and use it throughout the lesson.

How to get back to the prompt

If the command line hangs or you want to stop a process that is ongoing, use the keystroke Ctrl-C.

Sometimes you'll see it as ^C.

Try this out by using the following command:


            $ cat

Hint: It's waiting for your input...

Repeating and editing commands

Up-arrow cycles through commands you've typed previously in the same session.

Left-arrow and Right-arrow let you move around the line and edit what you've typed.

The command isn't submitted until you hit Enter/ Return.

You could also demo using:

  • Ctrl-A (move to the beginning of the line)
  • Ctrl-E (move to the end of the line)
  • Ctrl-U (deleting everything to the left of the cursor)

sudo

Use with caution!

This command, short for "super user do", appended before you enter any other command, gives you root privileges.

That is, you can execute any command, including modifying sensitive system files.


            $ sudo gem install bundler

We don't have `sudo` in Glitch, but it's useful to cover in case students see it online in tutorials, documentation, or SO answers.

Directories

A directory is what you know as a folder.

You can see your current working directory (the one you are "in") with pwd.

You can change your working directory with cd.


              $ pwd
              $ cd directory_name

Directory shortcuts

Several special characters are used for shortcuts in navigating the file structure. You'll see them used often in paths.

The single dot . represents the current directory.

The double dot .. represents the parent directory.

The forward slash / by itself represents the root directory.

The tilde ~ represents the home directory.

Try these out for yourself!


              $ cd ..
              $ cd ../..
              $ cd /
              $ cd ~
              $ cd

Consider drawing a file tree on the whiteboard to help students visualize how things are related and nested. This is similar to the DOM tree they saw earlier in class.

The home directory on Mac OS is /Users/username, and on Linux it is /Home/username.

View Directory Contents


            $ ls
            $ ls -l
            $ ls -a
            $ ls -al

Have students try these out to see what they do.

Make and Remove Directories


            $ mkdir new_directory
            $ rmdir empty_directory
            $ rm -rf directory_full_of_stuff # watch out!

Make sure to warn students about being careful with that last command! There is no undo if you accidentally delete your root directory.

Clearing the terminal

Sometimes the screen can feel cluttered. To clear your terminal screen you can:

  • Type clear at the prompt
  • Command-K on Mac OS

This doesn't clear your command history; it just makes your screen look neater!

Working with Files


              $ touch new-file.txt
              $ open index.html
              $ cp index.html main.html
              $ mv index.html ~/projects/website/index.html
              $ rm new-file.txt

Go over what these commands do and the way arguments work with these commands: first arg is origin, 2nd arg is destination.

Use man pages for reference.

Paths

Spaces and capitalization DO matter!

Don't put spaces in your file or directory names -- they have to be escaped when typed on the command line.


            $ cd My\ Project\ Folder
            $ cd my_project_folder

Relative and Absolute Paths

Paths indicate the "address" of a file or directory.


              $ cd projects/ruby/blackjack
              $ cd /Users/myusername/code/projects/ruby/go-fish
              $ cd ~/Documents/bash_commands.txt
              $ cd ../study_notes

Relative paths are relative to the current working directory.

Absolute paths are relative to the root directory and begin with "/".

Ask students to recall the directory shortcuts covered earlier. What do these paths lead to?

Copy and Move Directories


              $ cp -R projects portfolio
              $ mv projects ~/momentum/portfolio
              $ mv -i oldname newname

The mv command is useful for renaming files and directories. The -i option is a safeguard against an accidental overwrite.

Viewing File Contents


              $ cat ~/.bash_profile

Opening a File

You can open the default text editor, or any desktop app on a Mac, from the command line.


              $ open -t newfile.txt
              $ open -a Atom index.html

Cool CLI tricks

Word Count

Count the lines, words, and characters in a file.

Try these out and see what they do!


            $ man wc
            $ wc -l file.txt
            $ wc -m file2.txt
            $ wc -w file3.txt

Some things to try...

Don't forget to check man pages for each of these to find out how to really use them!


            $ date
            $ cal
            $ whoami
            $ uptime
            $ history
            $ say hello $(whoami)

Processes & disk space

Check out what your CPU is working on and what your hard disk memory looks like!


            $ top
            $ ps
            $ du

Processes are instances of programs the computer is running in memory. Each process has a process id (PID) that you can see and access if you need to.

Students may have seen this type of information before if they have looked at Task Manager or Activity Monitor.

Have students try these out with you. They will need to use Crtl-C to get out of top.

Piping

You can send the output of one command as input to the next command using the pipe operator:


            $ history | less
